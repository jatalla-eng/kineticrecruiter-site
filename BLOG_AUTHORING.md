# Blog Authoring Playbook

**Who this is for:** Any Claude Code (or human) session asked to write, edit, or expand a blog post on `kineticrecruiter.com`. Read this before touching anything in `content/blog/`.

---

## TL;DR

1. Scaffold the file: `npm run new-post -- my-slug "Optional Title"`
2. Write the content in `content/blog/<slug>.md` following the structure below.
3. Drop a hero image at `public/images/blog/<slug>.jpg`.
4. `git add content/blog/<slug>.md public/images/blog/<slug>.jpg && git commit && git push origin main`.
5. Cloud Build deploys, CDN purges, IndexNow pings Bing — all automatic.

No manual sitemap update. No manual GSC/Bing resubmission. The sitemap is generated by `src/app/sitemap.ts` on every build; IndexNow is pinged as a post-deploy Cloud Build step (`scripts/ping-indexnow.js`).

---

## Deploy pipeline (what happens after `git push`)

1. **Cloud Build** triggers on push to `main` (see `cloudbuild.yaml`).
2. **Docker build** → **Artifact Registry** → **Cloud Run** rollout.
3. **CDN invalidation** — global `kineticrecruiter-urlmap` purge (`--path=/*`, `--async`).
4. **IndexNow ping** — Node script fetches the live `sitemap.xml` and POSTs every URL to `api.indexnow.org`. Bing / DuckDuckGo / Yandex pick up new posts within minutes. Google ignores IndexNow; it discovers via sitemap recrawl in ~24 hr.

**Rollback:** `git revert <sha>` and push. The same pipeline handles rollbacks.

## Frontmatter spec (every field matters for SEO)

```yaml
---
title: "Primary Keyword or Working Title — 50-60 chars preferred"
date: "YYYY-MM-DD"                              # publish date; stays fixed once published
category: "Recruitment"                         # Recruitment | Comparisons | AI | Product | How-to
description: "Meta description, 150-160 chars. Include the primary keyword, the specific value, and a reason to click. This is the snippet Google shows in SERPs — treat it like ad copy."
image: "/images/blog/<slug>.jpg"                # OG image; must exist in public/images/blog/
author: "KineticRecruiter Team"                 # or a real named author for E-E-A-T signal
---
```

### Field rules

- **`title`** — stays under 60 chars to avoid SERP truncation. Front-load the keyword. Colons and em-dashes are fine.
- **`date`** — use real current date for new posts. Do not backdate to fake recency; Google catches it.
- **`category`** — must match one of the categories the Blog index filters by (check `src/components/blog/BlogIndex.tsx` if adding a new category).
- **`description`** — 150-160 chars, keyword in the first 100 chars. Do not copy the opening paragraph.
- **`image`** — 1200×630 JPG or WebP preferred. Used for OG/Twitter cards and the blog card thumbnail. If missing, the default OG image (`/images/og-default.jpg`) is used but the blog card renders blank.
- **`author`** — leave as `KineticRecruiter Team` by default. Swap to a real author name when writing signed authority pieces (founder POVs, technical deep-dives).

## Content structure (the shape every post should follow)

```markdown
---
<frontmatter>
---

**Quick answer:** One or two sentences that directly answer the query. AI search engines (Perplexity, ChatGPT search, Google SGE, Bing Copilot) lift this paragraph verbatim as a citation. Make it self-contained, concrete, and opinionated. No hedging. No "it depends" — it always depends, say the opinion anyway.

## First H2 heading (the actual answer, expanded)

Body copy in prose paragraphs. 2-4 sentences per paragraph. Short paragraphs are fine. Long walls of text are not.

Use bullet lists **only** for:
- Comparing features between options
- Enumerating steps in a process
- Listing distinct items (tools, criteria, red flags)

Do **not** bullet-list prose ("Here are the reasons: • because... • because..."). Write prose as prose.

## Second H2 (usually the nuance / trade-offs)

More depth. Specific numbers, proper nouns, comparisons to named competitors. Concrete examples beat abstractions every time.

## Third H2 (usually the "so what" / practical advice)

What should the reader do with this information? Be specific.

---

**Ready to see it in action?** [Start your free trial](https://app.kineticrecruiter.com/register) — no credit card, 14 days, full access.
```

### Length

- **Comparison / alternatives posts** (`*-alternatives-<year>.md`, `*-vs-*.md`): 1800–2800 words. These convert and rank.
- **How-to / explainer posts**: 1200–1800 words. Focus on specificity.
- **Opinion / POV posts**: 800–1500 words. Shorter is fine if sharp.

**Minimum threshold:** 800 words. Anything shorter looks thin to Google and has little chance of ranking.

---

## SEO conventions (non-negotiables)

1. **Primary keyword appears in:** title, H1 (same as title), first H2, URL slug, first 100 chars of body, and meta description.
2. **Related / LSI keywords** naturally throughout — don't stuff, but do mention synonyms and related terms a human expert would use.
3. **Internal links:** every post links to at least 2 other posts (relevant ones, not random) and 1 product page (`/`, `/pricing`, `/features/*`, `/compare/*`). Use descriptive anchor text, not "click here."
4. **External links:** link out to authoritative sources when citing stats/research. Helps E-E-A-T.
5. **Schema:** blog posts automatically get `BlogPosting` schema via the Next.js layout. Do not inline JSON-LD.
6. **URL structure:** slugs are kebab-case, keyword-first, no dates in the URL (dates go in frontmatter only — keeps URLs evergreen). Example: `best-ats-for-recruitment-agencies-2026` not `2026-04-best-ats`.

## AI-search optimization (the "Quick answer" pattern)

AI search engines parse structure. To maximize the chance they cite the post:

- **Open with `**Quick answer:**` in bold.** One paragraph. Self-contained. Names specific products/tools where relevant. No "it depends" — commit to a position.
- **Use H2 for major sections, H3 for subsections.** Don't skip levels.
- **Answer the question in the first 300 words.** Don't bury the lede.
- **Include a comparison table** when comparing 3+ options. HTML table or a `## Summary` section at the end.
- **Include an FAQ section** if the topic has obvious follow-up questions. The existing blog slug page renders FAQs as an accordion if structured with H3 question headings.

---

## Voice and style

- **Direct, concrete, opinionated.** Write like a subject-matter expert talking to a peer, not a marketing writer talking down to a prospect.
- **Name specific products and competitors.** Don't say "other ATS platforms" — say "Bullhorn, JobAdder, and JobDiva."
- **Specific numbers over vague claims.** "40% of JobAdder's typical cost" beats "much cheaper."
- **US market voice.** The site pivoted from en-AU to en-US in `46e0f3f`. Write for US recruiters: dollars, US time zones, US agency sizes, US staffing terminology.
- **No emojis.** No exclamation marks except in direct quotes.
- **No "here's why that matters"** or similar filler meta-commentary.
- **No bullet-listed prose.** If a bullet is a full paragraph, it's prose, not a bullet.

---

## Images

- **Hero image:** 1200×630 JPG/WebP at `public/images/blog/<slug>.jpg`. Generate via the `/design-generate-image` workflow (Nano Banana) or pick a relevant stock photo. Must have descriptive alt text (set in the frontmatter rendering logic — check `src/app/blog/[slug]/page.tsx`).
- **Inline images:** not required, but if used, compress to <200 KB each. Store at `public/images/blog/<slug>-<descriptor>.jpg`.
- **Do NOT:** use animated GIFs (hurts LCP), screenshots with unredacted personal data, or images with embedded text that will be misread by OCR.

## Reference posts to study before writing

Pick the most similar pattern to what you're writing and match its structure:

| Pattern | Reference post | Use when... |
|---|---|---|
| **Alternatives / competitor swap** | `content/blog/bullhorn-alternatives-2026.md`, `content/blog/jobadder-alternatives-2026.md` | Targeting "X alternatives" queries |
| **Head-to-head comparison** | `content/blog/vincere-vs-kineticrecruiter-2026.md` | Targeting "X vs Y" queries |
| **Buyer's guide / listicle** | `content/blog/best-ats-for-recruitment-agencies-2026.md` | Targeting "best X for Y" queries |
| **Explainer / how-it-works** | `content/blog/ai-candidate-scoring-explained.md`, `content/blog/semantic-search-vs-keyword-search-recruiting.md` | Targeting "how does X work" or "X vs Y" technical queries |
| **Tactical how-to** | `content/blog/boolean-search-cheat-sheet-recruiters.md` | Targeting "how to X" instructional queries |
| **Trend / industry POV** | `content/blog/ai-changing-recruitment-agencies-2026.md`, `content/blog/grow-recruitment-agency-with-ai.md` | Thought-leadership and topline awareness |

**Always mimic the structure of the closest reference.** Don't invent new shapes.

## What NOT to do

- **Do not** paraphrase or quote competitor marketing copy. Describe their product factually in your own words. Copyright / IP risk.
- **Do not** invent statistics. Every number needs a source (linked or at least attributable to a real report).
- **Do not** use `'en-AU'` in any new code you touch — the site is en-US. (See `46e0f3f` for context.)
- **Do not** add `use client` to the blog slug page or related components — they are server components.
- **Do not** install new fonts, animation libraries, or CSS frameworks. See `CLAUDE.md` "What NOT to Do."
- **Do not** edit `src/app/sitemap.ts` — it auto-discovers posts from `content/blog/`.
- **Do not** manually ping IndexNow for a single post — the deploy pipeline does it with the full sitemap.
- **Do not** commit images to `public/images/.originals/` — that folder is gitignored (pre-optimization backups).

## Post-publish checklist (optional, for important posts)

After push + deploy (`~5 min`):

1. Open `https://kineticrecruiter.com/blog/<slug>` in incognito, verify it renders and the OG image loads (paste the URL into https://www.opengraph.xyz for a quick check).
2. Run the URL through **GSC → URL Inspection → Request Indexing** for instant Google crawl (optional; Google will find it in 24 hr via sitemap anyway).
3. If it's a cornerstone post, add internal links to it from 2-3 relevant existing posts.
4. For competitor-comparison posts, add a row to `/compare` if the competitor isn't already listed.

## Quick reference — the commands

```bash
# Scaffold
npm run new-post -- my-slug "Optional Title"

# Publish
git add content/blog/my-slug.md public/images/blog/my-slug.jpg
git commit -m "blog: <short description>"
git push origin main

# Verify after deploy (~5 min)
curl -sI https://kineticrecruiter.com/blog/my-slug | head -5
```
