# KineticRecruiter SEO Audit

**Date:** 2026-04-17
**Site:** https://kineticrecruiter.com
**Stack:** Next.js 16 (App Router) + Tailwind v4 · Cloud Run + LB + CDN (AU SE1)
**Audited scope:** Full site — 21 URLs in sitemap (14 core pages + 7 blog posts)

---

## Executive Summary

The technical foundation is **good but incomplete**. SSR, metadata, sitemap, robots.txt, per-page titles/descriptions, canonicals, and JSON-LD are mostly in place and well-structured. However, there are **four blocking-level issues** that will actively hurt rankings and measurement, plus a set of high-impact fixes that should ship within 1–2 weeks.

### Top 5 priorities
1. **Zero analytics / Search Console verification in code** — no GA4, Plausible, PostHog, Clarity, or GSC verification tag. You have no way to measure SEO.
2. **Blog posts have no `<link rel="canonical">`** — `generateMetadata` in [blog/[slug]/page.tsx](src/app/blog/[slug]/page.tsx#L17-L33) omits `alternates.canonical`.
3. **Footer links to 404 pages** (`/about`, `/docs`) — crawlers will follow these.
4. **USD pricing on an AU-targeted site** — no hreflang `en-AU`, pricing schema in USD, domain is `.com`.
5. **Thin blog (7 posts) + no competitor-alternative pages** — major missed PSEO opportunity for AU agency keywords.

### Quick wins (<1 day each)
- Add canonical to blog posts
- Fix `/about` and `/docs` footer links (remove or ship pages)
- Update footer copyright from 2024 → 2026
- Compress oversized PNGs in `/public/images` (5+ MB each)
- Verify domain in Google Search Console + submit sitemap

## 1 · Crawlability & Indexation

### 1.1 Robots.txt — PASS
[public/robots.txt](public/robots.txt) correctly allows all crawlers (including GPTBot, ClaudeBot, PerplexityBot, Applebot-Extended, Google-Extended) and references the sitemap. Clean.

### 1.2 XML Sitemap — PASS with minor issues
[src/app/sitemap.ts](src/app/sitemap.ts) is properly generated at `/sitemap.xml`, returns 21 URLs, and updates blog post `lastmod` from frontmatter.

| Issue | Impact | Fix |
|---|---|---|
| Homepage URL ends with `/` but canonical omits trailing slash | Low — Google normalises, but inconsistency | Change [sitemap.ts:18](src/app/sitemap.ts#L18) to `https://kineticrecruiter.com` (no trailing `/`) |
| `/privacy` and `/terms` exist (200) but absent from sitemap | Very Low — legal pages don't need to rank | Optional: add with `priority: 0.3` |

### 1.3 Redirects — PASS with 1 cosmetic issue
- `http://` → `https://` ✅
- `www.` → root: `301 → https://kineticrecruiter.com:443/` — works, but the explicit `:443` port in Location is **non-standard**. Some legacy crawlers and SERP validators flag this. Strip the port.
- Trailing slash: `/pricing/` → `/pricing` via `308` ✅

### 1.4 Canonicals — FAIL (blog posts)
Homepage and every feature/solution/tool/compare/roi/blog-index page emit a correct self-referencing canonical. **Blog post pages emit none** — verified by fetching `/blog/best-ats-for-recruitment-agencies-2026` (zero `<link rel="canonical">` tags).

**Root cause:** [blog/[slug]/page.tsx:17-33](src/app/blog/[slug]/page.tsx#L17-L33) builds `generateMetadata` by hand instead of using `generatePageMetadata()`.

**Fix:**
```tsx
// src/app/blog/[slug]/page.tsx
import { generatePageMetadata } from '@/lib/metadata';

export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const { slug } = await params;
  const post = await getPostBySlug(slug);
  if (!post) return { title: 'Post Not Found' };
  return generatePageMetadata({
    title: post.title,
    description: post.description,
    path: `/blog/${slug}`,
    image: post.image,
    type: 'article',
    // title suffix: consider dropping "Blog" from " | KineticRecruiter Blog" —
    // use the standard " | KineticRecruiter" suffix to stay under 60 chars
  });
}
```

### 1.5 404 pages — PASS
[not-found.tsx](src/app/not-found.tsx) sets `robots: 'noindex, follow'` and returns a proper 404 (confirmed on `/about` and `/docs`).

### 1.6 HTTPS / SSL — PASS
Valid cert covers root + www. HSTS not verified but LB terminates TLS.

## 3 · Performance & Core Web Vitals

### 3.1 Cache headers — PASS
CDN 600s, browser must-revalidate, ETag present, `x-nextjs-prerender: 1` (SSG). TTFB 209ms. Good.

### 3.2 Hero is `'use client'` — MONITOR
[Hero.tsx:1](src/components/sections/Hero.tsx#L1) is a client component with three `useEffect` animations, IntersectionObserver, typing animation, and requestAnimationFrame counters. The H1 still SSRs so **LCP element is text** — should be fine. But the hydration bundle will be large. Run Lighthouse/PSI against the live homepage and check INP.

**Recommendation:** extract the static copy + H1 into a server component; keep only the animated browser mockup as the client island. Saves hydration cost on the LCP-critical above-the-fold region.

### 3.3 Fonts — PASS
`next/font/google` (Inter) handles preload/swap. No font CSS in globals.

### 3.4 No Core Web Vitals telemetry
Without RUM or GA4, you can't see real-world LCP/INP/CLS. See §6.

## 4 · Content & Keyword Coverage

### 4.1 Blog is thin — 7 posts

Only 7 blog posts total ([content/blog/](content/blog/)). For a B2B SaaS in a search-competitive niche (ATS, AI recruiting), this is well below the content velocity needed to win long-tail traffic. You should be publishing 2–4 posts/month minimum.

Current post titles are well-chosen (match intent, use `2026` freshness, cover transactional + informational). Build on them:

**Missing topic clusters (priority order):**
1. **Competitor alternatives** — `[competitor] alternatives`, `[competitor] vs KineticRecruiter`. The `/compare` page is one mega-page; **split into programmatic pages** (see 4.2).
2. **Australian-specific content** — "Best ATS for Australian recruitment agencies", "ATS compliance Australia", "Privacy Act recruitment". Domain is `.com` but market is AU — lean into it.
3. **Bottom-of-funnel transactional** — "ATS pricing comparison", "ATS with free trial", "cheapest ATS for small agencies".
4. **How-to / Tool-adjacent** — "How to write a job description" (pairs with your JD generator tool), "Resume parsing explained", "Semantic search for recruiters" (already have one).
5. **Case studies / data posts** — EEAT signal; original data wins rankings.

### 4.2 Programmatic SEO opportunity — HIGH VALUE

You currently have **one** `/compare` page covering Greenhouse + Lever. The AU agency ATS buyer searches for:
- `bullhorn alternatives`
- `jobadder alternatives` (huge in AU)
- `vincere alternatives`
- `recruitcrm alternatives`
- `manatal alternatives`
- `workable alternative`
- `kineticrecruiter vs [competitor]`

**Recommendation:** build `/compare/[competitor]` dynamic routes with a data-driven template (competitor → feature matrix → pricing comparison → verdict CTA). This is exactly what wins SaaS SEO — and Next.js App Router makes it trivial. Use the **competitor-alternatives** skill for the template structure.

### 4.3 Australian localization — currently absent

Per project memory, this is a marketing site for Australian recruitment agencies, but:
- No `<link rel="alternate" hreflang="en-AU" href="https://kineticrecruiter.com/">` in the root layout
- `SoftwareApplication` offers use `priceCurrency: 'USD'` ([page.tsx:29](src/app/page.tsx#L29)) — inconsistent with AU target
- No GBP/international sites to disambiguate — so strictly speaking hreflang is optional, but at minimum the `Organization` schema should include `address` with `addressCountry: AU`, and pricing should clarify currency (or show AUD + USD toggle).

### 4.4 Tool page = underused SEO asset

`/tools/job-description-generator` is a free tool with high search volume ("free job description generator", "AI job description") but:
- No `SoftwareApplication` or `HowTo` schema on the page
- Tool page H1 is just "Recruiter Tools" (generic)
- Only one tool listed; `/roi` exists but isn't in `/tools` index

Free-tool marketing ranks very well if optimized. Use the **free-tool-strategy** skill.

## 5 · Measurement Gaps

**Zero analytics code found in the entire `src/` tree.** Grep returned no matches for gtag, GA_MEASUREMENT, plausible, umami, posthog, mixpanel, hotjar, clarity, intercom, drift.

**You cannot run an SEO strategy without:**
1. **Google Search Console** verification (add meta tag to `layout.tsx` or DNS TXT record) — queries, impressions, CTR, crawl errors, index coverage, CWV real-user data.
2. **GA4 or Plausible/PostHog** — landing page performance, trial signups from organic, goal conversions.
3. **Bing Webmaster Tools** — ~7% of AU search, free data.
4. **Optional but high-ROI**: Microsoft Clarity (free heatmaps + session recordings) on high-intent pages (`/pricing`, `/compare`).

**Wire at minimum:** GSC verification + GA4 + event for "Start Free Trial" clicks, measured against page/source.
